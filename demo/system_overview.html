SCHEDULING ALGORITHM

每天撈題的
雙軸平衡演算法

雙層優先級 + 雙軸赤字計分，確保每月 30 篇按比例分配，且最久沒用的 seed 優先被撈。

📋 撈題優先級（兩層）

  1. scheduled FIFO：你 Discord 🟡 排明天的條目，先丟先發，status=used 一次性消耗
  2. normal 雙軸平衡：PDF seed + 🟢 存進的條目，循環使用不消耗

⚖️ 雙軸計分（normal）

  1. 算每軸本月已用幾篇 → 跟配額比 → 算赤字
  2. 主軸赤字最大的優先（A 13 / B 5 / C 3 / D 5 / E 3 / F 5）
  3. 副軸赤字次之（F1 11 / F2 9 / F3 7 / F4 3）
  4. 同分時 last_used_at NULLS FIRST ASC（最久沒用的優先）

🔄 永遠不會「發完」

  1. normal 條目用完不消耗，只更新 last_used_at
  2. 13 條 F seed → 約 2.5 個月跑完一輪
  3. 第 2 輪同 seed 被撈 → AI 重新生 caption + 33 slot，內容不重複
  4. 故事核心不變但寫法變化（hook、段落、CTA、版型色感都不同）

🆕 新增 seed 的入口

  1. Discord「ig｜文案寶藏庫」頻道丟訊息/圖/PDF
  2. Bot Haiku 4.5 自動分析（圖/PDF 升 Sonnet 4.6 視覺）
  3. 回 8 欄位預覽 → reply 1 存 / reply 2 排明天 / reply 4 修
  4. 進靈感庫後加入循環
